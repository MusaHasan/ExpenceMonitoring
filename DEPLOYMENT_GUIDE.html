üìã Prerequisites

1 Node.js & npm

2 Git Repository

Ensure your project is pushed to a GitHub repository.

üí° Note: All deployment methods require your code to be in a Git repository, preferably on GitHub.

üåê GitHub Pages Deployment

Deploy your app directly to GitHub Pages using GitHub Actions (Recommended)

Automatic Deployment (GitHub Actions)

1 Verify GitHub Actions Workflow

Ensure the file .github/workflows/deploy.yml exists in your repository.

2 Push to Main Branch

Push your code to the main or master branch to trigger the deployment.

git push origin main
3 Enable GitHub Pages

Go to your repository Settings ‚Üí Pages and set the source to "GitHub Actions".

4 Access Your App

Your app will be available at: https://yourusername.github.io/your-repo-name

‚úÖ Success! Your app is now live on GitHub Pages and will automatically update on every push to the main branch.

Manual Deployment

1 Run Deployment Script 
chmod +x deploy.sh
./deploy.sh
2 Upload to GitHub Pages

Manually upload the contents of the dist folder to your GitHub Pages branch.

‚ö° Vercel Deployment

Deploy to Vercel for lightning-fast performance and automatic deployments.

1 Install Vercel CLI 
npm install -g vercel
2 Deploy to Vercel 
vercel --prod
3 Follow the Prompts

Vercel will ask you to log in and configure your project. The vercel.json file will handle the configuration automatically.

‚úÖ Success! Your app is now live on Vercel with automatic deployments on every push.
üí° Pro Tip: You can also connect your GitHub repository to Vercel for automatic deployments without using the CLI.

üîß Troubleshooting

Common Issues & Solutions

‚ö†Ô∏è Build Fails

Problem: npm run build fails with errors

Solution:

# Clear cache and reinstall
rm -rf node_modules package-lock.json
npm install
npm run build
‚ö†Ô∏è Port Already in Use

Problem: Development server can't start

Solution:

# Kill processes on ports
npx kill-port 5173
npx kill-port 3001
‚ö†Ô∏è GitHub Actions Fails

Problem: GitHub Actions workflow fails

Solution:

  • Check the Actions tab in your repository
  • Verify Node.js version in workflow file
  • Ensure all dependencies are in package.json
‚ö†Ô∏è Routing Issues

Problem: Page refreshes show 404 errors

Solution: Ensure your hosting service is configured for SPA routing (all routes redirect to index.html)

Getting Help

üìû Support Resources:

  • Verify all dependencies are installed correctly
  • Ensure both frontend and API servers are running (if using API mode)

  • Open an issue in the repository for additional help
